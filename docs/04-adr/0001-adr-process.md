# ADR-0001: Registro y formato de ADRs

Estado: accepted
Fecha: 2026-03-05

## Contexto
Necesitamos registrar decisiones arquitectónicamente significativas en el repositorio (docs-as-code) para evitar ambigüedad y “conocimiento en chats”.

Referencia (origen de ADRs): https://www.cognitect.com/blog/2011/11/15/documenting-architecture-decisions

## Decisión
- ADRs en `/docs/04-adr/`
- Formato: Contexto / Decisión / Alternativas / Consecuencias
- Estados: proposed/accepted/superseded/deprecated
